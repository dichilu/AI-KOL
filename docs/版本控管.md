# 版本控管

## 版號規則

格式：`v主版本.次版本.修補`

| 碼 | 意義 | 觸發條件 |
|---|---|---|
| 主版本 | 架構級變動 | 整個流程重構、換平台、砍掉重練 |
| 次版本 | 功能級變動 | 新增 Agent、新增 Tab、Dashboard 新功能 |
| 修補 | 細節級變動 | 修 Bug、調整欄位、改 Prompt 措辭 |

`v0.x.x`：開發階段，功能尚未正式上線
`v1.0.0`：正式部署，行銷人員可開始使用
